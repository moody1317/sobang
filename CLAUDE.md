# CLAUDE.md — sobang (관할구역 위험 브리핑 시스템)

소방대원용 위험 브리핑 대시보드. 출동 데이터·건물 정보·날씨를 기반으로
관할구역 위험도를 스코어링해 브리핑, 위험 지도, 점검 리스트를 제공한다.

## 구조
- `client/` — React + Vite 프론트엔드 (dev 서버: 5173)
- `server/` — Express.js API 서버 (포트 3000). 모든 라우트는 `/api` 프리픽스
- `ml/` — Python + scikit-learn 위험도 스코어링

## 명령어
- 프론트 실행: `cd client && npm run dev`
- 백엔드 실행: `cd server && npm run dev`
- 검증: `cd client && npm run lint && npm run build`

## 코드 정확도 (MUST)
- 존재를 직접 확인하지 않은 함수·컴포넌트·파일 경로·API는 사용 금지. 사용 전에 해당 파일을 먼저 읽어라.
- 라이브러리 API를 쓰기 전에 `package.json`(Python은 `requirements.txt`)에서 설치된 버전을 확인하라. 그 버전에 없는 API를 추측해서 쓰지 마라.
- 새 로직을 작성하기 전에 같은 역할의 코드가 이미 있는지 grep으로 검색하라. 있으면 재사용하고 새로 만들지 마라.
- 확신이 없으면 추측으로 코드를 쓰지 말고, 관련 파일을 읽거나 사용자에게 질문하라.
- 모든 변경 후 검증 명령을 실행하라. 에러가 남아 있으면 작업 완료가 아니다.
- 실행이나 테스트로 동작을 확인하지 못한 코드는 완료 보고에 "미검증"이라고 명시하라.

## 불필요한 코드 금지 (NEVER)
- 요청받지 않은 기능·리팩토링·"개선" 추가 금지. 요청한 것만 최소 diff로 구현하라.
- 사용하지 않는 import, 변수, 함수, props, CSS 클래스 금지.
- 요청 없는 try/catch, null 방어 체크, fallback 로직 금지. 필요해 보이면 코드에 넣지 말고 완료 보고의 "제안" 섹션에만 적어라.
- 한 곳에서만 쓰는 로직을 함수·컴포넌트·유틸로 추상화하지 마라. 두 번째 사용처가 생겼을 때 분리하라.
- 자명한 코드에 주석 금지. 주석은 "왜"에 대한 설명이 필요한 곳에만 작성하라.
- `console.log`, 주석 처리된 죽은 코드 등 디버깅 잔해는 커밋 전에 제거하라.

## 프로젝트 규칙 (MUST)
- API 호출은 상대경로 `/api/...`만 사용. 백엔드 URL 하드코딩 금지 (Vite proxy가 `vite.config.js`에서 :3000으로 포워딩한다).
- `vite.config.js`의 proxy 설정 수정 금지.
- 색상·간격은 CSS 변수만 사용, hex 직접 입력 금지. 메인 컬러는 `--color-primary` (#B5333D 계열).
- 아이콘은 Bootstrap Icons만 사용.
- 새 컴포넌트·라우트는 같은 폴더의 기존 파일 구조를 그대로 모방해서 작성하라.

## Git
- `git push --force` 금지.
- 새 패키지 설치 전에 사용자에게 확인받아라.
- `.env`와 API 키는 절대 커밋하지 마라.
